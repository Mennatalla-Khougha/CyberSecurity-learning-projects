Phase 01 — Foundations
// DURATION: 6–8 WEEKS · DIFFICULTY: BEGINNER
Build the bedrock. Before hacking anything you need to understand how systems, networks, and Linux internals actually work. Your DevOps background is a huge head start. This phase turns your existing knowledge into a security-aware mindset.
P-1.1
Build Your Hacking Lab RESUME
BEGINNER
▼
Goal: Set up a safe, isolated environment on your Linux machine using VMs where you can practice attacking and defending without risk. This becomes your lab that you'll use for the rest of the roadmap.
// TOPICS & MILESTONES
1. Virtualization & Networking Setup
Understand how VMs work, how to create isolated networks, and how traffic flows between them. You'll use KVM/QEMU (native Linux) or VirtualBox.
  • Install VirtualBox or configure KVM/QEMU on Ubuntu
  • Create an isolated "host-only" network adapter
  • Spin up Kali Linux VM (attacker) and Metasploitable2 VM (victim)
  • Verify VMs can ping each other but NOT reach the internet
  • Snapshot both VMs so you can restore after experiments
✓ ACCEPTANCE CRITERIA

Kali can ping Metasploitable2. Metasploitable2 cannot reach 8.8.8.8. Both VMs are snapshotted. You can write a brief network diagram of your lab in your notes.

2. Linux Security Hardening Basics
Learn to think like a defender first — understand file permissions, users, sudoers, SSH keys, firewall (ufw/iptables), and audit logs.
  • Audit your Ubuntu host: list all open ports (ss -tulpn), running services, SUID binaries
  • Create a non-root user, set up SSH key auth, disable password login, disable root SSH
  • Configure ufw: deny all incoming, allow only SSH from specific IP
  • Read /var/log/auth.log and identify failed login attempts
  • Write a bash script that automates this hardening checklist
✓ ACCEPTANCE CRITERIA

Your hardening script runs cleanly. SSH only accepts key auth. UFW is active. You can explain what each rule does. You have a written checklist of 10+ security items you audited.

3. Python for Security Scripting
Write small security tools. Python is the lingua franca of security. You'll build a port scanner from scratch — not using nmap, but using raw sockets.
  • Build a TCP port scanner using Python sockets (scan a range of ports)
  • Add threading to make it fast
  • Add banner grabbing (connect and read what the service says)
  • Save output to a JSON report file
  • Test it against your Metasploitable VM
✓ ACCEPTANCE CRITERIA

Your scanner can scan 1-1024 ports on Metasploitable in under 30 seconds, grabs banners from open ports, and outputs a JSON report. Code is on GitHub.

✓ ACCEPTANCE CRITERIA

2. Packet Analysis with Wireshark & tcpdump
Understand what's actually happening on the wire. Every attack generates traffic. Learn to read it.
  • Capture traffic between your two VMs using Wireshark
  • Filter and identify: ARP, DNS, HTTP, FTP, Telnet packets
  • Capture an FTP login on Metasploitable — extract the plaintext credentials from the capture
  • Use tcpdump from the command line to capture and filter traffic by port/protocol
  • Write a Python script using Scapy to craft and send a custom TCP packet
✓ ACCEPTANCE CRITERIA

You can identify a protocol from a packet capture in under 30 seconds. You've extracted real credentials from an FTP capture. Your Scapy script successfully sends a SYN packet and captures the SYN-ACK.

  • Exploit Command Injection: get a reverse shell to your Kali machine
  • Exploit File Upload: upload a PHP webshell, execute commands
  • For each: write the vulnerable code snippet AND the fix
✓ ACCEPTANCE CRITERIA

Phase 04 — Defense, Blue Team & Detection
// DURATION: 6–8 WEEKS · DIFFICULTY: INTERMEDIATE
The best attackers understand defense. This phase leverages your DevOps background heavily — you'll build a SIEM, create detection rules, and learn to hunt threats. Blue Team skills are in massive demand and often pay better than pure red team.
P-4.1
Build a Home SIEM Lab RESUME
INTERMEDIATE
▼
Goal: Deploy a functional SIEM (Security Information & Event Management) in your lab using free tools. Configure log ingestion, write detection rules, and trigger real alerts from attacks you run yourself.
1. ELK Stack (Elasticsearch, Logstash, Kibana)
ELK is the most widely deployed free SIEM stack. Understanding log aggregation, parsing, and visualization is a core Blue Team skill and directly applies to your DevOps experience.
  • Deploy ELK Stack on a dedicated Ubuntu VM using Docker Compose
  • Configure Filebeat on Kali to ship logs to ELK
  • Configure Winlogbeat on a Windows VM (free trial) to ship Windows Event Logs
  • Ship auth.log, syslog, and apache logs to ELK
  • Build a Kibana dashboard: login failures, SSH activity, web traffic
  • Run a brute-force attack against your lab → observe it in Kibana in real-time
✓ ACCEPTANCE CRITERIA

ELK is live and ingesting logs from 2+ sources. Your Kibana dashboard shows SSH login events. When you run hydra against your Ubuntu VM, the failed logins appear in Kibana within 30 seconds. Dashboard is screenshot-worthy for your portfolio.

📡 FREE RESOURCES
ELK Stack Download (free) Elastic Getting Started Guide TryHackMe ELK 101 Room TryHackMe: ItsyBitsy (ELK lab)
2. Detection Engineering — Write Sigma Rules
Sigma is a generic signature format for SIEM rules — like Snort but for logs. Writing detection rules is a highly sought skill. You'll write rules to detect the attacks from Phase 3.
  • Learn Sigma rule format (yaml-based)
  • Write a Sigma rule to detect: SSH brute force (5+ failed logins in 60 seconds)
  • Write a Sigma rule to detect: successful login after multiple failures (spray attack)
  • Write a Sigma rule to detect: nmap SYN scan (detect by log patterns)
  • Convert your Sigma rules to Elastic/Kibana queries using sigma-cli
  • Test each rule — trigger it manually and confirm alert fires
✓ ACCEPTANCE CRITERIA

3 working Sigma rules committed to GitHub. Each rule has a test procedure documented. When the attack is executed in your lab, the alert fires in Kibana. Rules follow Sigma community standards.
